# 🎨 QuickStart Vibe — Para CEO / Founder / Criador
**Leitura: 5 minutos** | **Pré-requisito: zero conhecimento técnico**

> Você vai aprender a construir coisas (posts, agentes, módulos) descrevendo o que quer em linguagem natural. **Sem comandos. Sem código. Sem medo.**

---

## 1. 🪄 Como funciona (em uma frase)

> **Você descreve o que quer → A IA executa → O Foundry garante que ficou certo.**

Você fala com o Claude Code normalmente, do jeito que falaria com uma pessoa. Por trás, o Foundry invisivelmente:
- Verifica se sua ideia faz sentido (alguém vai pagar?)
- Calcula o custo antes de fazer
- Cria os testes
- Documenta tudo
- Te mostra o resultado

Você nunca precisa saber esses detalhes. Só precisa **confiar** que estão acontecendo.

         [preview das 5 imagens + legenda]

### Exemplo 2 — Quero criar um agente de atendimento

**Você digita:**
> "Quero um agente que responda as mensagens do Instagram automaticamente 24h por dia"

**O que vai acontecer:**

```
🎨 Foundry: "Ótimo! Antes de construir, preciso entender 3 coisas:
         
         1. Quem é o cliente ideal para responder? (ex: industrial, varejo)
         2. Qual o tom de voz? (formal, amigável, técnico)

       perguntarem preço ou quando o cliente reclamar"

         Você vai poder ver o agente respondendo de teste antes de soltar

```

## 3. 📚 Glossário Leigo (só o essencial)

Você vai ouvir esses termos. Tradução:

| Termo técnico | O que significa de verdade |
|--------------|--------------------------|
| **Foundry** | As regras invisíveis que evitam que você quebre coisas |
| **Constitution** | As 8 regras-mãe que NUNCA são quebradas (tipo "não cobre sem saber se dá margem") |
| **Agente** | Um trabalhador digital que faz uma tarefa específica (designer, copywriter, etc) |
| **Guardian** | Um inspetor que verifica se cada agente fez certo |
| **SKU / Produto** | Algo que você vende/entrega para um cliente |
| **Outcome** | O resultado concreto que o cliente paga (ex: "carrossel publicado") |
| **SHADOW / Pilot** | Modo "teste sem cobrar" antes de soltar pra valer |
| **AUTONOMOUS / Canonical** | Modo "produção, pode cobrar" |
| **Eval** | Os testes que garantem que o agente funciona |
| **Unit economics** | Quanto você gasta vs quanto cobra (margem) |
| **Spec** | O documento que descreve o que cada agente faz |

**Atalho mental:** quando o agente disser uma sigla esquisita (C3, C4, ADR, etc), você pode responder:
> "Me explica isso em português que eu não sou técnica"

E ele vai traduzir.

## 4. 🎯 5 Receitas Práticas

### Receita 1 — "Quero criar conteúdo"

```
Você: "Crie [tipo de conteúdo] sobre [tema] para [rede social]"

Exemplos:
• "Crie um post de carrossel sobre vendas para LinkedIn"
• "Crie 5 stories sobre lançamento do produto X"
• "Crie um Reel de 30 segundos sobre IA para indústria"
```

### Receita 2 — "Quero criar um agente"

```
Você: "Quero um agente que [tarefa] para [público]"

Exemplos:
• "Quero um agente que qualifique leads no WhatsApp"
• "Quero um agente que analise concorrentes toda segunda"
• "Quero um agente que gere relatórios semanais para os clientes"
```

### Receita 3 — "Quero criar um módulo de plataforma"

```
Você: "Quero adicionar [funcionalidade] no [sistema]"

Exemplos:
• "Quero adicionar área de financeiro no CRM"
• "Quero adicionar exportação de relatório no painel"
• "Quero adicionar login com Google no app"
```

### Receita 4 — "Quero entender o que aconteceu"

```
Você: "Me mostra o que foi feito na última semana"
Você: "Por que [coisa específica] está [problema]?"
Você: "Qual o custo atual de [agente/módulo]?"
```

### Receita 5 — "Quero refazer/ajustar"

```
Você: "Refaz, mas [ajuste específico]"
Você: "Pode trocar [coisa A] por [coisa B]?"
Você: "Tira [elemento] e coloca [elemento novo]"
```

## 5. 🚦 Sinais de "Tá tudo bem" vs "Espera aí"

### ✅ Sinais de que tá tudo bem:
- O agente confirmou o que vai fazer antes
- Mostrou custo e tempo estimados
- Pediu sua aprovação antes de executar
- Mostrou preview do resultado
- Perguntou se quer ajustar

### ⚠️ Sinais de que precisa parar:
- O agente começou a fazer sem perguntar
- Você não entendeu o que ele falou
- Apareceu uma mensagem de erro em inglês ou com siglas
- O custo está acima do que faz sentido para você
- Você se sentiu pressionada a aprovar

**Se qualquer dos ⚠️ aconteceu:**
> "Para. Me explica em português o que está acontecendo."

O agente vai parar e traduzir.

## 6. 🆘 Quando der errado

### "Apareceu um erro técnico que não entendi"

Digite:
> "Me explica esse erro em linguagem leiga e me diz o que fazer"

### "O agente está rodando há muito tempo"

Digite:
> "Para o que está fazendo e me mostra onde parou"

### "Quero desfazer o que foi feito"

Digite:
> "Me ajuda a desfazer a última ação"

> ⚠️ Importante: o Foundry **não deixa** o agente deletar coisas importantes sozinho. Mesmo que você peça "apaga tudo", ele vai confirmar 2x antes.

### "Não sei se posso confiar no que ele fez"

Digite:
> "Mostra um resumo do que foi entregue e o que ainda falta"

## 7. 🎁 Bônus: Como pedir bem

**Pedidos que funcionam melhor:**

✅ "Crie um post de carrossel sobre IA para indústria, 5 slides, tom the CEO"
✅ "Quero entender quanto custou nosso conteúdo do mês passado"
✅ "Refaz o slide 3, mas sem o ícone de robô"

**Pedidos que confundem o agente:**

❌ "Faz alguma coisa" (muito vago)
❌ "Sei lá, vê aí" (sem direção)
❌ "Tipo o que a the CEO postou outro dia" (sem contexto)

**Truque:** sempre que pedir, inclua:
1. **O que** quer (carrossel, agente, módulo)
2. **Para que público** (industrial, varejo, etc)
3. **Em qual canal** (Instagram, LinkedIn, etc)
